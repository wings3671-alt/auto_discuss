## 项目一句话

一个本机命令行调度器:让 Claude Code、Codex 与用户本人围绕一份 Markdown
文档自动一轮一轮交替发表意见,把议题讨论深、讨论透。脚本只做调度、不生成
内容;参与者只走各自订阅、绝不走 API 计费。

### 1b · 上下文优化(2026-05-17)
**为什么做**:原本每轮把讨论 md 全文喂给 AI,累计 token 平方级增长(成本、
延迟、长讨论里注意力被稀释)。目标:每轮喂进的量是个小常量,不随轮数增长。

**第一步——先单独定一轮 STATE_OPS 格式**(commit `1a9adea`,存为 `1b_STATE_OPS规格.md`):
- 关键决策:状态更新指令用 **JSON**,不用自定义中文命令 —— 有标准解析器、
  可做 schema 校验。
- 滚动白板 `STATE.md` 四栏目;便签编号 C/Q/R/K **单调递增、永不复用**;由程序
  独家维护。
- 操作只 3 种 `add` / `update` / `remove`;严格校验、整批原子应用。

**实现时撞到的关键岔路——「讨论文档由谁执笔」**:
- 甲(交接文档原案):AI 自己把发言写进讨论 md。问题:AI 追加前通常要先读
  全文 → token 又被载入它的上下文 → 1b 省 token 的目标落空。
- 乙(采纳):AI **不碰文件**,只把「发言 + STATE_OPS」作为回复输出;**程序
  执笔**写讨论 md 并更新 STATE.md。→ 省 token 目标稳达成;AI 碰不到文件,
  「改坏前文」风险消失;程序成为两个文件唯一的笔。
- 此改动偏离了交接文档字面(「AI 只写讨论 md」),经与用户确认后采纳乙。

**实现与调整**(commit `0ba2788`):
- AI 用**只读模式**调用(claude 不开 acceptEdits;codex 用 read-only 沙箱),
  可读文件回查、不可写。
- STATE_OPS 不合格 → 重试 1 次 → 仍不行则**软兜底**(跳过本轮白板更新、留
  可见警告、讨论继续),不中断整场讨论。
- 小试点后用户反馈:讨论 md 里那段 STATE_OPS JSON 碍眼 → 改成发言末尾**一行
  人话摘要**(如「📋 本轮白板更新:新增 2 条约束」),JSON 不再写进文档。
  (commit `fde69b3`)
- 小试点:2 轮三方讨论,4 个 AI 回合共 16 条 STATE_OPS 全部首次合法;白板
  正确滚动(含 AI 用 `update` 修正自己上一轮写的条目);token 用量两轮持平,
  验证「不随轮数膨胀」达成。

## 关键设计决策(升级前必读)

1. **计费铁律**:只走 Claude / Codex 各自订阅,绝不走 API。调子进程前剔除
   `*_API_KEY`;CLI 用本机已登录账号。
2. **三个文件各司其职**:讨论 md(人读 + 完整记录)、`STATE.md`(滚动白板,
   程序独家维护)、文档顶部 `AUTO_DISCUSS` 状态块(轮次 / turn,程序独家维护)。
3. **程序执笔(1b 方案乙)**:三方模式下 AI 不写文件、程序写。这是 1b 省 token
   能成立的关键 —— 升级时**不要**退回「AI 自己写文件」。
4. **白板是被增改的清单,不是被重写的文章**:STATE_OPS 三种操作即够用;编号
   单调不复用,是防 AI 引用到被回收的旧号、改错便签。
5. **用户永远压轴、且是决策者**:只有用户能终结讨论;用户发言全文传给 AI、
   绝不压缩。
6. **`--no-user` 旧两方路径完整保留**,1b 只改三方默认路径。
7. **安全机制全程保留**:每轮时间戳备份、文件锁、原子写、防篡改回滚。

## 已踩过的坑(交接文档第 7 节之外的补充)
- 嵌套环境变量污染:调 `claude` 前必须剔除 `CLAUDE*` / `AI_AGENT`,否则它误以为
  自己嵌套在另一个会话里、空跑秒退。
- 改发言标题格式时,**校验正则必须同步改**,否则误判本轮无产出。
- 标记词(`over`/`收工`/`FINAL_DONE`)只在「本轮新增区域 / 最新用户段」内、按
  「独立整行」匹配,防止正文别处的同名字样误触发。
- 双击运行的 `.command` **不加载 shell 配置** → 启动器脚本里要自己补 `PATH`
  与 UTF-8 环境(`PYTHONUTF8` / `LANG`),否则找不到 python3、或子进程中文乱码。
- 连本地服务(如 Ollama localhost)要**显式绕开代理** —— `urllib` 会读
  `http_proxy` 环境变量,把 localhost 请求也塞给代理导致 502。
